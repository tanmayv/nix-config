Mastering the Pi TUI

What is the Pi TUI?

The Pi Terminal User Interface (TUI) is a powerful, component-based system that lets you build rich, interactive experiences directly in your terminal. It's designed for extensions and custom tools to present users with dialogs, selectors, loaders, and even custom editors.

Key Features

  • Component-Based: Build complex UIs using standard blocks like Text, Box, and Container.
  • Responsive Overlays: Easily spawn modal dialogs or side panels that handle their own positioning and visibility.
  • Theme Awareness: Components automatically adhere to Pi's current color theme via a provided theme object.
  • Advanced Input: Full support for keyboard events, IME for CJK, and even custom input logic (perfect for Vim-like modes).
  • Media Support: Render Markdown and images directly in the terminal (Kitty, iTerm2, WezTerm).

How to Extend Pi

  1. Create a Component: Implement the Component interface (render, handleInput, invalidate).
  2. Use the API: Use ctx.ui.custom() in extensions or pi.ui.custom() in tools to render your component.
  3. Follow Patterns: Don't reinvent the wheel! Use built-in helpers:
    • SelectList for picking options.
    • BorderedLoader for async tasks.
    • SettingsList for toggles.
    • CustomEditor for replacing the main input.

Development Pro-Tips

  • Performance: Always cache your rendered output and use invalidate() to trigger updates.
  • Safety: Use truncateToWidth() to ensure your UI looks great on any terminal size.
  • Theming: Never hardcode colors. Always use the theme object provided in the render context.
